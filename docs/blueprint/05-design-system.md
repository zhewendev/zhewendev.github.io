> 定义网站的设计哲学、视觉语言及交互原则。本文档描述设计语言，具体数值参数见 `07-design-tokens.md`。

# 一、文档目标

建立统一、稳定的设计语言，保证：

- 统一的视觉表达与交互体验；
- 不同页面具有一致的阅读节奏；
- 新增页面、组件有明确的设计依据；
- 设计能够长期演进，而非随页面增加不断改变风格。

# 二、设计哲学与核心原则

> **让设计退居幕后，让内容成为主角。**

最高原则：任何设计决策都应回答——它是否帮助用户更轻松地阅读内容？若否，则不加入。

### 设计哲学

| 哲学 | 含义 |
|------|------|
| 内容优先（Content First） | 设计服务于内容理解，而非吸引注意力 |
| 长期主义（Timeless） | 不追逐短期流行趋势，五年后仍具良好阅读体验 |
| 克制（Calm） | 设计保持安静，不主动争夺注意力 |
| 精致（Elegant） | 来自对齐、留白、比例、排版，而非元素的丰富 |
| 可持续（Sustainable） | 新增页面优先复用已有设计语言，而非重新设计 |

### 核心原则

> 通用原则见 `00-overview` §原则与理念。以下为设计领域的特定原则。

1. **阅读优先**：任何设计都不应增加阅读成本。不为美观降低对比度、不因动画增加等待、不因布局压缩正文宽度。
2. **一致优先**：同类元素始终统一表现（卡片、标签、分页、导航）。用户无需重新学习。
3. **减少认知负担**：用户始终清楚身在何处、可做什么、当前内容属性。避免复杂导航、重复入口、信息堆叠。
4. **层级清晰**：层级来自标题、排版、留白、位置、字号，而非颜色、阴影、边框、动画。
5. **渐进增强**：首先保证内容完整，其次提升视觉体验，最后增加交互体验。

# 三、品牌人格

像一位长期学习、乐于分享的工程师：**理性、温和、可信、克制、持续成长**。

- **气质关键词**：Calm、Rational、Warm、Trustworthy、Thoughtful、Minimal、Timeless。
- **情绪表达**：保持平稳，避免过度兴奋、夸张表达、营销式文案、煽动性语言。
- **文案语气**：像经验丰富的朋友分享实践，而非老师授课或广告宣传。
- **图标与插画**：图标为辅助，插画为氛围，二者不担任页面主角。优先用真实内容建立吸引力。

# 四、阅读体验与视觉语言

所有设计服务于长时间阅读。视觉语言克制、稳定，主要依赖排版、留白、比例、对齐、节奏。

- **阅读节奏**：正文保持稳定节奏，避免广告、推荐、弹窗、动画等干扰。
- **视觉焦点**：任何时刻存在唯一焦点——正文。导航、目录、标签、推荐均为辅助。
- **信息密度**：保持适中，避免过空、过满、过密、过散。
- **阅读连续性**：长文通过目录、上下篇、专题导航、阅读进度帮助用户建立上下文。
- **视觉层次**：每个视觉元素承担明确职责，无纯装饰元素。留白属于内容的一部分，建立节奏与自然分隔。全站统一对齐方式，建立视觉秩序。

# 五、交互原则

交互帮助用户理解状态变化，而非制造惊喜。自然、克制、可预期。

- **快速确认**：每次操作立即获得反馈。
- **克制过渡**：动画仅用于理解状态变化（Hover、展开、收起、页面切换、深浅色切换）。避免突然出现/消失、明显跳动、强烈缩放、复杂弹跳。
- **减少干扰**：阅读过程避免自动弹窗、自动播放、自动滚动、自动展开、营销提示、持续浮动元素。所有交互由用户主动触发。
- **一致反馈**：相同行为始终产生相同反馈。
- **渐进呈现**：默认展示当前最重要的信息，复杂信息逐步展开。

# 六、布局系统

- **页面结构**：统一 Header → Main → Footer。
- **内容区域**：始终最高优先级，不压缩正文空间。具体宽度参数见 `07-design-tokens.md` §8。
- **页面层级**：页面 → 区块 → 组件 → 内容。避免页面直接堆叠大量组件。
- **区块组织**：通过留白、排版、层级建立边界，而非边框、背景、颜色。
- **页面节奏**：不同页面内容不同，但浏览节奏一致。

# 七、排版系统

排版是最重要的视觉设计。具体字号阶梯、字重、行高见 `07-design-tokens.md` §3。

- **标题层级**：承担信息导航，层级明确区别。仅浏览标题即可理解文章结构。
- **正文**：始终最高可读性——字号适中、字重适中、颜色清晰、段落舒适、行长适度。
- **列表**：表达并列/步骤/归纳关系，每个列表具备明确语义。
- **引用**：强调来源而非观点，明显区别于正文但不破坏阅读节奏。
- **代码**：属于正文，优先保证完整、清晰、易复制、易阅读。
- **表格**：表达结构化信息，优先完整、对齐清晰、浏览效率。
- **图片**：服务于正文，帮助解释内容，不承担装饰。

# 八、色彩哲学

颜色表达信息，非装饰。低饱和、稳定、克制。具体色板见 `07-design-tokens.md` §1。

- **中性色优先**：页面主体以中性色构建。品牌色仅用于强调，状态色仅表达状态。
- **品牌色**：克制使用，仅用于链接、交互、强调、当前状态。
- **状态色**：仅表达成功、警告、错误、提示，不承担视觉设计职责。
- **对比关系**：清晰、舒适、易阅读，颜色之间建立稳定层级。

# 九、组件原则

- **单一职责**：一个组件只解决一个问题（如 ArticleCard 仅展示文章，不负责搜索/分页/推荐）。
- **可组合**：组件自由组合，页面由组件构成，而非组件依赖页面。
- **一致性**：同类组件统一表现（Card、Badge、Pagination、Button）。
- **内容驱动**：组件展示内容，不创造内容，不拥有业务逻辑。
- **可替换**：允许重写组件，不影响内容、页面、布局。

# 十、复杂内容承载

知识内容不可避免包含长文、大量代码、大型表格、流程图、引用、嵌套列表。当复杂性与简洁性冲突时，优先保证内容表达。

- **内容优先**：页面首先保证内容完整，其次阅读体验，最后视觉整洁。不为美观删减内容。
- **大段代码**：允许占据更多空间、横向滚动、折叠辅助内容。默认完整展示。
- **大型表格**：保持完整，根据设备调整展示方式，不隐藏重要数据。
- **长文阅读**：通过目录、阅读进度、专题导航、上下篇降低阅读负担。
- **信息密度自适应**：长文/短文/代码文章/读书笔记允许不同信息密度，布局骨架稳定。

# 十一、响应式设计

天然适配不同设备。目标是所有用户拥有一致的阅读体验，而非完全一致的布局。具体断点见 `07-design-tokens.md` §7。

- **阅读体验一致**：桌面/平板/移动端内容体验一致。允许导航形式、栏目排列、卡片数量变化，不允许信息/功能缺失。
- **内容优先**：小屏优先保留正文、标题、导航、搜索，辅助内容逐步弱化。
- **自然适应**：根据可用空间自然调整布局，不针对特定设备单独设计页面。

# 十二、暗色模式哲学

暗色模式是阅读环境，目标是帮助用户在低光环境下持续阅读，而非营造科技感。具体切换机制见 `07-design-tokens.md` §14。

- **保持一致性**：延续日间模式设计语言，仅调整亮度、对比、视觉重心。不变更页面结构、组件样式、信息层级。
- **降低视觉疲劳**：背景不追求绝对纯黑，正文不追求绝对白。整体柔和稳定。
- **内容仍是主角**：切换后视觉焦点不变，正文仍最高优先级。

# 十三、无障碍设计

遵循最低限度的无障碍要求，确保内容对所有用户可读、可操作。

- **语义化 HTML**：使用正确的语义标签（`nav`、`main`、`article`、`aside`、`section`），不滥用 `div`。
- **键盘可访问**：所有交互元素可通过键盘操作，焦点状态清晰可见（项目已通过 `focus-visible:outline` 实现）。
- **对比度**：正文与背景的对比度满足 WCAG AA 标准（≥ 4.5:1）。
- **替代文本**：内容性图片提供 `alt` 描述，装饰性图片使用空 `alt`。
- **跳转链接**：提供 skip-to-content 链接，允许键盘用户跳过导航。

上述要求不增加额外开发成本，仅是良好 HTML 实践的基线。

# 十四、一致性与弹性

骨架一致保证学习成本低，内容弹性保证阅读体验。

- **骨架一致**：导航、布局、组件、设计语言全站统一。用户无需重新适应。
- **内容弹性**：正文内根据内容特点调整节奏，信息密度可不同。变化服务阅读，非追求视觉差异。
- **克制个性化**：允许适度表达内容特点，但不破坏整体设计语言。

# 十五、长期演进

- **稳定优于创新**：新增设计优先复用已有语言。只有在无法满足需求时才新增模式。
- **渐进优化**：允许持续改进，但每次调整保证整体一致、用户熟悉、内容稳定。
- **可维护**：任何新增设计应回答——是否长期可维护？是否可被其他页面复用？
- **AI 协作**：AI 设计页面/组件时遵循本设计系统，不为视觉效果破坏整体语言。

# 十六、设计决策优先级

多原则冲突时，按以下顺序权衡：

```text
阅读体验（Reading Experience）
    ↓
内容完整（Content Integrity）
    ↓
一致性（Consistency）
    ↓
可维护性（Maintainability）
    ↓
视觉美观（Visual Appearance）
    ↓
个性化设计（Decoration）
```

应用示例：

- ❌ 为页面整洁裁剪代码示例 → 不允许
- ❌ 为统一卡片高度隐藏文章摘要 → 不允许
- ❌ 为减少页面元素删除专题导航 → 不允许
- ✅ 为提升阅读体验增加正文留白 → 允许
- ✅ 为提高代码可读性增加滚动区域 → 允许
- ✅ 为移动端阅读调整布局 → 允许
