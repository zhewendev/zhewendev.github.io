> 定义网站的信息组织方式、页面结构及导航关系，是页面实现和功能开发的上游依据。

---

# 一、设计目标

内容优先，层级扁平。用户能快速理解结构、找到内容、以最少点击完成浏览，在阅读中自然发现相关内容。

整个站点保持层级简单、路径清晰，避免过深的页面嵌套。

---

# 二、站点地图

采用一级导航 + 内容列表 + 内容详情的组织方式，分类/标签页不出现在导航栏。

```text
首页

├── 文章（Articles）
│   ├── 文章列表
│   ├── 分类页（导航隐藏）
│   ├── 标签页（导航隐藏）
│   └── 文章详情
│
├── 专题（Series）
│   ├── 专题列表
│   └── 专题详情
│         └── 引用文章
│                ↓
│            文章详情（共用文章详情页）
│
├── 收藏（Collections）
│   ├── 书架
│   │     ├── 图书列表
│   │     └── 图书详情
│   │
│   └── 工具箱
│         ├── 工具列表
│         └── 工具详情
│
├── 随想（Thoughts）
│   ├── 时间线
│   └── 随想详情
│
├── 友链（Friends）
│
└── 关于（About）
```

说明：

- 一级导航仅展示六个核心栏目。
- 分类页、标签页属于文章系统，不出现在导航栏。
- 同一篇文章可以从文章列表或专题详情进入，但始终使用同一个文章详情页。
- 收藏中的书籍和工具拥有各自独立的详情页，以便支持 SEO、搜索及长期扩展。

---

# 三、导航设计

六个固定栏目，稳定不增。未来功能优先融入现有栏目，不新增一级导航。

| 栏目 | 定位           | 是否核心内容 |
| ---- | -------------- | ------------ |
| 文章 | 深度知识输出   | ✅            |
| 专题 | 系列化知识组织 | ✅            |
| 收藏 | 优质资源沉淀   | ✅            |
| 随想 | 轻量内容       | ✅            |
| 友链 | 博客交流       | 否           |
| 关于 | 个人介绍与导流 | ✅            |

---

# 四、页面职责

- **首页**：站点入口，非聚合页。展示定位、最新文章、精选专题。
- **文章列表**：所有长内容的统一入口，支持分类/标签筛选、搜索、分页。
- **文章详情页**：核心阅读页，展示正文、目录、阅读信息、分类/标签、所属专题、上下篇、推荐阅读。专题不生成独立文章页。
- **专题列表**：展示所有专题，仅展示专题简介与统计，不展示文章内容。
- **专题详情页**：组织学习路径，含简介、阅读建议、更新状态、有序文章列表。文章点击进入统一详情页。
- **收藏**：书架/工具箱各自拥有列表页和详情页，详情展示资源介绍、标签、推荐及关联文章。
- **随想**：时间线组织，独立详情页，不承担系统知识输出。
- **关于**：个人介绍、项目、联系方式、全平台入口。
- **友链**：独立博客链接交换。

---

# 五、页面流转关系

页面之间遵循以下访问关系。

- 首页 → 文章列表 → 文章详情
- 专题列表 → 专题详情 → 文章详情（共用）
- 收藏 → 书架/工具箱 → 详情页 → 关联文章
- 随想时间线 → 随想详情

---

# 六、搜索范围

覆盖文章、专题、收藏、随想。不搜索关于、友链。结果按内容类型分组展示。

# 七、URL 设计原则

不同内容类型拥有独立的 URL 空间。

建议：

```text
/articles/          → 列表
/articles/{slug}/   → 详情
/series/            → 列表
/series/{slug}/     → 详情
/collections/       → 收藏首页
/thoughts/          → 随想
/friends/           → 友链
/about/             → 关于
/categories/{slug}  → 分类筛选（导航隐藏）
/tags/{slug}        → 标签筛选（导航隐藏）
```

专题中的文章链接始终指向 `/articles/{slug}`，保证文章地址独立稳定，支持多专题引用。

---

# 八、未来扩展原则

未来新增功能时，应优先考虑放入现有栏目，而不是新增一级导航。

AI 搜索（全站能力）、项目展示（关于）、Newsletter（关于/文章页）、年度总结（收藏/关于）

---

# 九、信息架构原则

> 通用原则见 `00-overview` §原则与理念。以下为信息架构领域的特定原则。

1. 一级导航固定精简
2. 页面职责单一
3. 内容间显式关联
4. 降低寻找成本
5. 架构随内容增长保持稳定，不增层级
