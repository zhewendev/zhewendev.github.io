## 需求清单

### 1. 首页（Home `/`）

- **描述**：站点入口，非聚合页。展示定位、最新文章、精选专题、最近收藏、最近随想。
- **设计稿**：`G:\MistDoc\博客设计稿\首页\PC-首页-new.png` + 移动端版本
- **Blueprint 参考**：`01-information-architecture.md` §四（页面职责）、§七（URL 设计）；`05-design-system.md` 全文；`07-design-tokens.md` 全文

#### 区块 1：Header 导航栏
- **布局**：左 Logo + 站点名 | 中一级导航 | 右搜索/主题切换/更多菜单
- **导航项**（6 个固定栏目）：文章、专题、收藏、随想、关于（友链不在此处，可能放入更多菜单或 Footer）
- **右侧图标**：搜索图标、暗色模式切换按钮、更多菜单（⋯）
- **涉及组件**：Header.astro（改造现有）、SearchButton、ThemeSwitch、MenuButton
- **决策**：
  - 一级导航固定 6 项，与 Blueprint §三一致
  - 搜索入口在 Header 右侧，点击展开搜索（具体交互待确认，可能在搜索页讨论时确定）
  - 友链不在主导航中，可能放入 Footer 或更多菜单

#### 区块 2：Hero 首屏区
- **背景**：全宽背景图，支持多张轮换，图片来源为本地 `public/` 目录
- **主标题**：「记录思考，沉淀知识」
- **副标题**：「分享技术、阅读与生活的点滴，构建属于自己的知识宝库。」
- **CTA 按钮**：
  - 主按钮（实心）：「阅读文章」→ 链接到 `/articles`
  - 次按钮（描边）：「关于我」→ 链接到 `/about`
- **涉及组件**：Hero 组件（新建）、背景图轮换逻辑
- **开放问题**：
  - 轮换机制：自动轮播？每次刷新随机？手动切换？（建议简单方案：每次刷新随机选一张，避免自动轮播的复杂性和对阅读体验的干扰）
  - 背景图片尺寸规格和数量？

#### 区块 3：内容区 — 左栏「最新文章」（~2/3 宽）
- **展示数量**：默认 5 篇，可根据效果调整，上限 10 篇
- **分页**：首页内分页（不跳转到 `/articles`），分页器样式：`1 2 3 ... 10 下一页`
- **移动端**：不展示分页器，改为在「最新文章」标题右侧显示「查看全部 →」链接，点击跳转 `/articles`
- **文章卡片结构**：
  - 左侧：缩略图（圆角矩形），来源 Frontmatter `heroImage` 字段；未指定时使用默认占位图
  - 右侧信息区：
    - 文章标题
    - 标签（蓝色胶囊 Badge，显示第一个标签或分类？待确认 → **决策：显示标签，可多个**）
    - ~~阅读量~~ → **改为文章摘要**（2-3 行文字截断）
    - 设计稿中缺失摘要区域，需补充设计
- **涉及组件**：ArticleCard 组件（改造现有 Card 组件）、Pagination 组件
- **关键决策**：
  - 去掉阅读量功能
  - 新增文章摘要字段/取正文开头截断
  - 缩略图 fallback 到默认占位图

#### 区块 4：内容区 — 右栏（~1/3 宽）
- **子区块 A：「专题推荐」**
  - 展示数量：3 个
  - 排序：支持配置（最新 / 精选），通过站点配置控制
  - 卡片结构：缩略图 + 专题名称
  - 涉及数据：Series Content Collection

- **子区块 B：「最近收藏」（仅书籍）**
  - 展示数量：≤ 3 个
  - 仅展示书籍（Book），不展示工具（Tool）
  - 列表样式：圆点 + 名称（简洁列表，非卡片）

- **子区块 C：「最近随想」（新增，设计稿未体现）
  - 展示数量：≤ 3 个
  - 列表样式：同收藏（圆点 + 名称 或 简卡片）
  - 位置：最近收藏下方

- **涉及组件**：SeriesCard（小卡片）、CollectionListItem、ThoughtListItem
- **响应式**：移动端时整个右栏移到左栏下方

#### 区块 5：Footer
- **四列布局**：
  - 导航：首页、文章、专题、随想、关于
  - 资源：RSS 订阅、Sitemap、标签、分类、存档
  - 联系：GitHub、Email、微信公众号
  - 订阅更新：邮箱输入框 + 订阅按钮（**置灰禁用状态**，功能暂不实现，后续再定）
- **社交图标行**：GitHub、小红书、微信公众号、抖音、知乎、哔哩哔哩、Email、RSS
- **底部版权行**：© 2024 XiaoYu · 粤ICP 备 · RSS · Sitemap
- **涉及组件**：Footer.astro（大幅改造）、SocialIcons 组件（更新图标集）
- **关键决策**：
  - 订阅区域置灰，不做功能实现
  - 社交图标扩展至 8 个平台（含国内平台）

#### 响应式断点策略

- **平板/移动端（< lg）**：单栏布局，右栏内容移至左栏下方
- **移动端特殊处理**：分页器替换为「查看全部」链接

### 2. 文章列表页（`/articles/`）

- **描述**：所有长内容的统一入口，支持分类筛选、搜索、分页。
- **设计稿**：文章列表页 PC 端设计稿（用户已提供截图）
- **Blueprint 参考**：`01-information-architecture.md` §四（页面职责）、§七（URL 设计）；`02-content-model.md` §3.1（Article 定义）

#### 区块 1：页面标题 + 搜索
- **页面标题**：「所有文章」（`text-2xl` 或 `text-3xl`）
- **搜索框**：
  - 位置：标题右侧
  - 占位符：「搜索文章...」
  - 交互：回车触发搜索，调用 Pagefind 在当前页过滤显示结果
  - 范围：仅搜索文章，不搜索书籍/工具/随想等
  - 与 Header 全局搜索的区别：全局搜索覆盖全站内容类型，此处仅文章
- **涉及组件**：ArticleSearch 组件（新建，封装 Pagefind 搜索逻辑）

#### 区块 2：分类筛选 Tab 栏
- **布局**：横向 Tab 切换栏，位于标题下方
- **首项**：「全部」（默认选中，显示所有分类的文章）
- **后续项**：动态从 Article 内容的 Category 字段提取，非固定写死
- **样式**：选中态为蓝色下划线/高亮，未选中为普通文字
- **URL 格式**：`/articles/?category=<slug>` （如 `/articles/?category=技术`）
- **「全部」状态**：`/articles/` 或 `/articles/?category=全部`
- **涉及数据**：Article Content Collection 的 `category` 字段

#### 区块 3：文章列表
- **布局**：单栏全宽（与首页双栏不同，此处无侧边栏）
- **每页数量**：默认 10 篇，可调整，上限 20 篇
- **文章卡片**：与首页 `ArticleCard` **完全一致**的组件复用
  - 缩略图（左）+ 标题 + 标签（Badge）+ 日期 + 摘要（右）
  - ~~阅读量~~ 已统一去掉
- **排序**：按发布时间倒序（最新在前）
- **分页器**：与首页共用 `Pagination` 组件，样式一致 `1 2 3 ... 10 下一页`

#### 区块 4：Footer
- 与首页完全一致，复用 Footer 组件

#### 响应式策略
- **桌面端**：单栏列表，搜索框与标题同行
- **移动端**：搜索框移到标题下方独立一行；分类 Tab 支持横向滚动

#### 涉及文件变更（预估）
- `src/pages/articles/index.astro` — 新建文章列表页
- `src/pages/articles/[page].astro` 或分页逻辑 — 分页路由
- `src/components/ArticleSearch.astro` — 新建文章搜索组件
- `src/components/CategoryTabs.astro` — 新建分类 Tab 组件
- 复用：`ArticleCard.astro`、`Pagination.astro`、`Footer.astro`

#### 关键决策汇总
| 决策项 | 结论 |
|--------|------|
| 筛选维度 | Category（分类），非 Tag |
| 分类来源 | 动态从内容提取，非固定 |
| 搜索方式 | Pagefind 当前页过滤，回车触发 |
| 搜索范围 | 仅文章 |
| 阅读量 | 去掉，统一替换为日期+摘要 |
| 卡片组件 | 与首页复用同一 ArticleCard |
| 每页数量 | 默认 10，上限 20 |
| URL 格式 | `/articles/?category=<slug>` |

### 3. 文章详情页（`/articles/{slug}/`）

- **描述**：核心阅读页，展示正文、目录、阅读信息、分类/标签、所属专题、上下篇、推荐阅读。
- **设计稿**：文章详情页 PC 端设计草稿（用户已提供截图）
- **Blueprint 参考**：`01-information-architecture.md` §四（页面职责）；`02-content-model.md` §3.1（Article 定义）；`05-design-system.md` §十（复杂内容承载）；`08-seo-spec.md` 全文（文章 Meta/OG/结构化数据）

#### 区块 0：面包屑导航
- **格式**：首页 / 文章 / {文章标题}
- 链接：首页 → `/`，文章 → `/articles/`
- 样式：简洁文字链，用 `>` 或 `/` 分隔
- **涉及组件**：Breadcrumb 组件（新建）

#### 区块 1：文章头部（左栏）
- **文章标题**：`text-3xl` 或 `text-4xl`，大标题样式
- **元数据行**：
  - 发布于 {pubDatetime} · 更新于 {modDatetime} · 约 {readingTime} 分钟
  - ~~阅读量~~ 已去掉
  - 字数统计（Frontmatter 手动填写 `wordCount` 字段）
- **标签行**：蓝色 Badge 标签（与列表页一致），可点击跳转到该标签筛选页
- **涉及数据**：Article Frontmatter — `pubDatetime`、`modDatetime`、`readingTime`、`wordCount`、`tags`

#### 区块 2：封面大图（Hero Image）
- **位置**：标题区下方，正文上方
- **尺寸**：全宽（左栏宽度内），比卡片缩略图大
- **来源**：Frontmatter `heroImage` 字段，与文章列表卡片缩略图为同一张图
- **Fallback**：未指定时使用默认占位图（与卡片共用同一张默认图）
- **涉及数据**：Article Frontmatter — `heroImage`

#### 区块 3：正文内容（左栏）
- **排版**：使用 `.app-prose` 类（Tailwind Typography），遵循 `typography.css`
- **内容**：Markdown / MDX 渲染后的完整正文
- **支持元素**：标题层级(h1-h4)、段落、代码块(Shiki 双主题)、表格、引用、图片、列表、链接等
- **涉及布局**：PostLayout.astro（改造现有）

#### 区块 4：底部互动栏（左栏底部）
- **点赞（👍）**：纯前端展示，localStorage 存储点赞状态和计数；无需后端
- **评论（💬）**：展示置灰不可点击状态，显示评论数（预留字段），暂不接入评论系统
- **分享**：调用 Web Share API（`navigator.share`），不支持该 API 的浏览器隐藏或降级为复制链接
- **布局**：横向排列，图标 + 数字
- **涉及组件**：ArticleActions 组件（新建）
- **关键决策**：
  - 点赞：localStorage 持久化，按文章 slug 存储
  - 评论：UI 展示但功能禁用（置灰），等后续确定评论系统再启用
  - 分享：优先 Web Share API，优雅降级

#### 区块 5：右侧边栏（~1/3 宽）

**子区块 A：文章目录（TOC）**
- 自动从正文中提取 h2-h4 标题生成
- 带层级缩进
- 点击滚动到对应标题（锚点跳转）
- 当前阅读位置高亮（滚动监听）
- **涉及组件**：TableOfContents 组件（改造或新建）

**子区块 B：文章信息卡片**
- 发布日期
- 更新日期
- 阅读时长
- 字数统计
- 简洁信息展示，非表格形式

**子区块 C：相关文章**
- 推荐规则：同标签 + 更新时间与当前文章较近
- 展示数量：≤ 3 篇
- 卡片样式：简版（标题 + 可能的缩略图或仅标题）
- **涉及逻辑**：构建时或运行时根据标签匹配计算相关度

**子区块 D：标签**
- 与头部标签行一致的 Badge 列表
- 可点击跳转

#### 响应式策略
- **桌面端（≥ lg）**：双栏布局（左正文 2/3 + 右侧边栏 1/3），右侧边栏固定或 sticky
- **移动端（< lg）**：
  - 右侧边栏变为**可折叠浮层**（通过按钮触发展开/收起）
  - 目录（TOC）同样在浮层中
  - 浮层触发按钮位置：页面右下角或标题旁

#### SEO 相关
- 文章详情页是 SEO 重点页面，需完整实现：
  - `<title>` 格式：`{article.title} | {site.title}`
  - `<meta description>`：取文章摘要/正文开头
  - Open Graph：type 为 `article`，含 published_time、modified_time
  - OG 图片：自动生成或 Frontmatter 指定
  - 结构化数据（JSON-LD）：Article schema
  - 规范链接（canonical URL）

#### 涉及文件变更（预估）
- `src/pages/posts/[...slug]/index.astro` 或 `src/pages/articles/[slug]/index.astro` — 重写文章详情路由
- `src/layouts/PostLayout.astro` — 改造文章布局（双栏结构）
- `src/components/Breadcrumb.astro` — 新建面包屑
- `src/components/TableOfContents.astro` — 目录组件
- `src/components/ArticleActions.astro` — 新建互动栏（点赞/评论/分享）
- `src/components/RelatedPosts.astro` — 新建相关文章
- `src/components/ArticleInfo.astro` — 新建文章信息卡片
- 复用：`Footer.astro`、Header 中标签 Badge 样式

#### 关键决策汇总
| 决策项 | 结论 |
|--------|------|
| 阅读量 | 去掉 |
| 点赞 | 纯前端 localStorage |
| 评论 | UI 置灰禁用，暂不接入 |
| 分享 | Web Share API + 降级 |
| 封面图 | 与卡片 heroImage 同源同字段 |
| 默认图 | 与卡片共用同一张默认占位图 |
| 字数 | Frontmatter 手写 wordCount |
| 相关文章 | 同标签 + 时间相近，≤3 篇 |
| 移动端侧边栏 | 可折叠浮层 |

### 4. 专题列表页（`/series/`）

- **描述**：展示所有专题，仅展示专题简介与统计，不展示文章内容。
- **设计稿**：专题列表页 PC 端设计草稿（用户已提供截图）
- **Blueprint 参考**：`01-information-architecture.md` §四（页面职责）；`02-content-model.md` §3.2（Series 定义）

#### 区块 1：页面标题 + 搜索
- **页面标题**：「所有专题」
- **搜索框**：
  - 位置：标题右侧
  - 占位符：「搜索专题...」
  - 交互：回车触发，调用 Pagefind 在当前页过滤
  - 范围：仅搜索专题内容
- **涉及组件**：可复用 ArticleSearch 的模式或新建 SeriesSearch

#### 区块 2：专题卡片网格
- **布局**：CSS Grid 网格，桌面端 3 列，平板 2 列，移动端 1 列
- **每页数量**：默认 12 个（4x3），可根据美感调整，上限 20 个
- **卡片结构**：
  - 顶部：封面图（占卡片约 60% 高度，圆角）
  - 底部信息区：
    - 专题名称（标题）
    - 文章数（如 `12 篇文章`）
    - ~~阅读量~~ 已去掉，仅保留文章数
- **点击行为**：整个卡片可点击，跳转到 `/series/{slug}/`
- **封面图来源**：Frontmatter `heroImage` 字段；未指定时使用默认占位图
- **涉及组件**：SeriesCard 组件（新建，与首页右侧「专题推荐」的小卡片为不同尺寸变体）
- **涉及数据**：Series Content Collection — `title`、`heroImage`、关联的 Article 数量

#### 区块 3：分页器
- 复用 `Pagination` 组件，样式一致
- URL 格式：`/series/?page=2`

#### 排序规则
1. **置顶优先**：被设置为置顶（`featured` / `pinned`）的专题优先展示
2. **其次按最新更新**：根据专题中最新文章的更新时间排序
- **涉及数据**：Series Frontmatter 需支持 `pinned: boolean` 或类似字段

#### 响应式策略
| 断点 | 列数 |
|------|------|
| ≥ lg (1024px) | 3 列 |
| ≥ sm (640px) | 2 列 |
| < sm | 1 列 |

#### 涉及文件变更（预估）
- `src/pages/series/index.astro` — 新建专题列表页
- `src/components/SeriesCard.astro` — 新建专题卡片组件（网格版）
- 复用：`Pagination.astro`、`Footer.astro`

#### 关键决策汇总
| 决策项 | 结论 |
|--------|------|
| 卡片底部信息 | 仅专题名称 + N 篇文章 |
| 阅读量 | 去掉 |
| 搜索范围 | 仅专题 |
