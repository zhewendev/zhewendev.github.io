## Context

基于 AstroPaper 模板构建个人知识博客。当前模板页面与产品设计稿不符，需要重构 9 个核心页面，建立统一的组件体系和交互模式。

**技术栈约束**：
- Astro v7（SSG 模式）
- TypeScript（strict 模式）
- Tailwind CSS v4
- Pagefind（静态搜索）
- Content Collections（Markdown/MDX）

**设计稿来源**：本地设计稿文件（PC-首页-new.png 等）

## Goals / Non-Goals

**Goals:**
- 按照设计稿实现 9 个核心页面（首页、文章列表/详情、专题列表/详情、随想列表、书籍收藏、工具收藏、关于）
- 建立可复用的组件体系（Header/Footer/Card/Pagination/Modal）
- 实现响应式布局（桌面双栏/移动端单栏）
- 统一视觉风格和交互模式
- 全站去除阅读量功能

**Non-Goals:**
- 不改动内容模型（使用现有 Article/Series，新增 Book/Tool/Thought/About）
- 不引入新的外部依赖
- 不实现评论系统（仅 UI 占位）
- 不实现订阅功能（仅 UI 占位）

## Decisions

### 1. 组件复用策略
- **ArticleCard**：首页、文章列表、专题详情复用同一组件
- **Pagination**：所有列表页复用同一组件
- **CategoryTabs**：文章/书籍/工具列表页复用同一组件样式
- **Modal**：书籍/工具收藏共用基础 Modal 容器

### 2. 页面布局模式
- **双栏布局**（左 2/3 + 右 1/3）：首页、文章详情、随想列表、关于页
- **单栏布局**：文章列表、专题列表、专题详情
- **网格布局**：专题列表（3列）、书籍/工具收藏（2列）

### 3. 移动端适配
- **可折叠浮层**：文章详情 TOC、随想列表日历区
- **布局切换**：双栏 → 单栏，网格 → 减少列数

### 4. 数据来源
- **站点配置**（astro-paper.config.ts）：导航、社交链接、Hero 文案、做事卡片
- **Content Collections**：文章、专题、随想、书籍、工具、关于页内容
- **Frontmatter**：分类、标签、封面图、评分、状态等

### 5. Hero 背景轮换
- **方案**：每次刷新随机选一张（服务端渲染时随机）
- **原因**：简单可靠，无需客户端 JS，避免自动轮播干扰阅读

### 6. 搜索实现
- **方案**：各页面独立搜索框 + Pagefind 当前页过滤
- **原因**：静态站点，Pagefind 已集成，无需后端

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 组件复杂度增加 | 按 Phase 分步实现，先基础组件后页面 |
| 设计稿与实现偏差 | 每完成一个页面进行视觉对比检查 |
| 移动端交互复杂（浮层/折叠） | 使用 Tailwind 响应式类，渐进增强 |
| 默认占位图资源缺失 | Phase 1 优先准备占位图资源 |

## Migration Plan

无需迁移（新站点搭建），按 Phase 顺序实现：
1. Phase 1: 基础组件与布局
2. Phase 2: 核心页面（首页/文章）
3. Phase 3: 内容列表页（专题/随想）
4. Phase 4: 收藏页面（书籍/工具）
5. Phase 5: 关于页
